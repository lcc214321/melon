设计思路：
1、根据注解@RibbonClients自定义规则
2、灰度规则根据SPI可扩展
3、仅支持eureka注册中心灰度
4、每个应用实例配置指定eureka appGroupName
5、灰度请求：
   a.来源于网关（zuul、gateway）
     需要在网关处计算灰度匹配与否
     如果有灰度匹配，则根据灰度配置服务组选择对应的具体服务应用实例，并向下传递上下文
     如果没有灰度匹配，则选择正常组对应的具体服务应用实例，并向下传递上下文
   b.来源于直接访问（如xxl-job）
     则需要根据当前北访问具体服务应用实例所在组，构建上下文，并向下传递
6、支持灰度配置与更新
